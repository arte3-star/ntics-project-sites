# Fotografia — NTICS Projetos
> Brand Book v2.0 | Atualizado: Marco 2026

---

## Visao Geral

A fotografia da NTICS Projetos deve transmitir **impacto real, conexao humana e profissionalismo**. As imagens mostram pessoas transformando realidades, tecnologia a servico do social e ambientes que refletem sustentabilidade. O tratamento fotografico se integra a linguagem de blocos geometricos da marca.

---

## Contextos Fotograficos

### B2B / Corporativo
Imagens para materiais institucionais, propostas comerciais, site e relatorios corporativos.

**O que fotografar:**
- Profissionais em reunioes e workshops colaborativos
- Ambientes de escritorio modernos e organizados
- Telas com dashboards, dados e graficos (contexto de tecnologia)
- Apertos de mao, assinatura de parcerias
- Eventos corporativos e conferencias
- Equipes diversas trabalhando juntas

**Tom:** profissional, confiavel, moderno, colaborativo

### Projetos / Impacto Social
Imagens que documentam o impacto real dos projetos da NTICS.

**O que fotografar:**
- Pessoas aprendendo: salas de aula, oficinas, capacitacoes
- Comunidades em acao: reunioes comunitarias, mutiroes, eventos locais
- Criancas e jovens em atividades educativas
- Detalhes de maos trabalhando, escrevendo, construindo
- Expressoes genuinas de concentracao, alegria, descoberta
- Antes e depois de projetos (transformacao visivel)

**Tom:** humano, autentico, esperancoso, real (nao encenado)

### Institucional / Sustentabilidade
Imagens para comunicacao de valores, ESG e relatorios de sustentabilidade.

**O que fotografar:**
- Natureza: florestas, agua, biodiversidade, paisagens regeneradas
- Energias renovaveis: paineis solares, turbinas eolicas
- Agricultura sustentavel e permacultura
- Metaforas visuais: sementes brotando, caminhos, horizontes
- Detalhes de texturas naturais (folhas, agua, terra)

**Tom:** inspirador, esperancoso, grandioso, sustentavel

## Tratamento Fotografico

### Cores Vibrantes
- Manter cores **vivas e vibrantes** nas fotos
- Leve aumento de saturacao (+10-15%) quando necessario
- Nao dessaturar ou aplicar filtros frios/cinzentos
- Balanco de branco neutro a levemente quente

### Overlay com Blocos Geometricos
A integracao de fotos com a linguagem visual da marca usa **blocos semi-transparentes**:

**Exemplo — Banner com foto e bloco:**
```
+-----------------------------------------------+
|                                               |
|   Foto de equipe em projeto social            |
|                                               |
|        +---------------------------+          |
|        | Bloco Verde Regeneracao   |          |
|        | Opacidade 80%             |          |
|        |                           |          |
|        |  "1060+ projetos          |          |
|        |   de impacto social"      |          |
|        |                           |          |
|        +---------------------------+          |
|                                               |
+-----------------------------------------------+
```

**Exemplo — Composicao lateral:**
```
+-------------------------+------------------+
|                         |                  |
|   Foto (2/3 largura)   | Bloco Azul       |
|                         | Petroleo         |
|                         |                  |
|                         | Titulo e texto   |
|                         | em branco        |
|                         |                  |
+-------------------------+------------------+
```

**Exemplo — Multiplos blocos sobre foto:**
```
+-----------------------------------------------+
|                                               |
|   Foto de paisagem sustentavel                |
|                                               |
|   +--------+                                  |
|   | Verde  |   +--------+                     |
|   | 80%    |   | Rosa   |                     |
|   | Stat   |   | 75%    |                     |
|   +--------+   | Quote  |    +---------+      |
|                +--------+    | Amarelo |      |
|                              | 85%     |      |
|                              | CTA     |      |
|                              +---------+      |
+-----------------------------------------------+
```

### Regras de Overlay
- Opacidade dos blocos sobre fotos: **70% a 85%**
- A foto deve permanecer **parcialmente visivel** atraves do bloco
- Texto sobre bloco translucido deve ter contraste WCAG AA
- **NAO usar** overlays opacos (100%) que escondem a foto
- **NAO usar** filtros monocromaticos, sepia ou preto-e-branco sobre as fotos
- Blocos devem seguir as regras de composicao assimetrica (ver `elementos.md`)

## Do's — Boas Praticas

### Escolha de Imagens
- Preferir fotos com **pessoas reais** em contextos reais (nao stock generico)
- Buscar **diversidade**: genero, etnia, idade, acessibilidade
- Incluir **detalhes e close-ups** alem de planos abertos
- Fotos com **luz natural** sempre que possivel
- Expressoes **genuinas** — evitar sorrisos forcados ou poses artificiais
- Imagens que contem **historias** — acoes acontecendo, nao pessoas paradas

### Qualidade Tecnica
- Resolucao minima: 300dpi (impresso), 72dpi / 2x (digital)
- Foco nitido no sujeito principal
- Exposicao equilibrada (sem areas estouradas ou muito escuras)
- Proporcoes comuns: 16:9 (banners), 4:3 (cards), 1:1 (redes sociais)

### Integracao com a Marca
- Fotos com presenca de cores que harmonizem com a paleta NTICS
- Ambientes com tons verdes, azuis ou neutros funcionam melhor
- Fotos com espacos vazios (negative space) para posicionar blocos e texto

## Don'ts — O que Evitar

### Escolha de Imagens
- **NAO usar** fotos stock generico sem contexto (pessoas apontando para graficos vazios)
- **NAO usar** fotos com marcas de terceiros visiveis
- **NAO usar** fotos de baixa resolucao ou pixeladas
- **NAO usar** fotos com cenarios desordenados ou sujos
- **NAO usar** fotos que reforcem estereotipos
- **NAO usar** fotos sem consentimento das pessoas retratadas (especialmente criancas)

### Tratamento
- **NAO aplicar** filtros monocromaticos ou preto-e-branco
- **NAO aplicar** filtros de cor unica sobre toda a foto (duotone generico)
- **NAO usar** vinheta forte (escurecimento excessivo nas bordas)
- **NAO dessaturar** as cores da foto
- **NAO adicionar** bordas, molduras ou efeitos vintage
- **NAO usar** overlays opacos que escondem a foto completamente

### Composicao
- **NAO cobrir** rostos de pessoas com blocos ou texto
- **NAO usar** fotos sem espaco para blocos/texto quando a composicao exigir
- **NAO distorcer** a proporcao da imagem (esticar/comprimir)

## Acessibilidade em Imagens

### Texto Alternativo (Alt Text)
Toda imagem deve ter texto alternativo descritivo:

- **Descrever o que a imagem mostra**, nao o que ela "significa"
- Incluir contexto relevante: "Grupo de jovens em oficina de robotica no projeto NTICS em Curitiba"
- Para imagens decorativas (engrenagens, blocos), usar `alt=""` (vazio)
- Maximo de 125 caracteres para alt text

### Contraste de Texto sobre Imagem
- Texto diretamente sobre foto: **evitar** (usar bloco translucido como intermediario)
- Bloco sobre foto com texto: verificar contraste WCAG AA (4.5:1 minimo)
- Se a foto for muito clara, usar bloco com cor escura (Azul Petroleo, Grafite)
- Se a foto for muito escura, usar bloco com cor clara e texto Grafite

### Conteudo Sensivel
- Fotos com criancas: obter autorizacao de imagem dos responsaveis
- Nao expor situacoes de vulnerabilidade de forma exploratoria
- Mostrar impacto positivo, nao o problema — foco na transformacao

## Banco de Imagens — Diretrizes para Producao

### Briefing para Fotografos
Ao contratar producao fotografica, incluir:

1. **Contexto:** tipo de projeto, local, publico
2. **Paleta de referencia:** enviar paleta NTICS para harmonizacao
3. **Composicao:** prever espacos para blocos geometricos
4. **Diversidade:** garantir representatividade nas pessoas fotografadas
5. **Iluminacao:** preferir luz natural, evitar flash direto
6. **Enquadramento:** capturar planos abertos E detalhes/close-ups
7. **Autorizacao:** coletar termos de uso de imagem

### Organizacao do Acervo
- Nomear arquivos com padrao: `ntics_[contexto]_[descricao]_[data].jpg`
- Exemplo: `ntics_projeto_oficina-robotica_2026-03.jpg`
- Manter metadados EXIF com creditos do fotografo
- Classificar por contexto: B2B, Projetos, Institucional
- Revisar acervo semestralmente para remover imagens desatualizadas
