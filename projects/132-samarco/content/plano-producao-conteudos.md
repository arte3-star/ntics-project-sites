## 5. Como iniciar uma produção

Para qualquer item 🟢 da seção 2:

1. **Definir briefing curto** (1 parágrafo, com referência a esta lista).
2. **Identificar assets-base** no [INDEX.md](../../../../AUTOMAÇÕES/assets/projetos/132. ESTAÇÃO SAMARCO/INDEX.md) da biblioteca.
3. **Disparar a skill correspondente** do plugin `ntics-brain` (ex: `/google-slides-template` para templates editáveis, `/post-instagram` para post único, `/carrossel-cliente` para carrosseis).
4. **Aprovação interna NTICS:** Bruna + Lucas obrigatórios.
5. **Aprovação Samarco:** Amanda + Rayane via Teams/SharePoint.
6. **Registro no `historico.md`** do projeto pela skill `/projeto-registrar`.

## 6. Conteúdos sugeridos como "primeiros 5"

Se quisermos um pacote inicial mínimo para validar o pipeline visual antes de escalar:

1. **Card editável "Inscrições abertas Brumal".** Template Google Slides com campos {DATA INÍCIO}, {LOCAL}, {LINK INSCRIÇÃO}.
2. **Post estático "Conheça a Estação Samarco".** 1 imagem 1080×1350, com logo, paleta extraída e personagem chef-mulher como hero.
3. **One-pager A4 dos dois cursos.** Frente e verso, com identificação de cada curso, carga horária, módulos e CTA.
