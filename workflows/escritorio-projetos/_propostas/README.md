# Propostas de skills

> Stubs gerados automaticamente por `/projeto-retrospectiva` quando o user responde "sim, vira skill nova" na pergunta (c).

## Como usar

- Cada proposta é um arquivo `.md` com o seed do que a skill deveria fazer.
- Para **promover** uma proposta a skill formal: rodar `/skill-creator` apontando para o arquivo.
- Para **descartar:** mover para `_descartadas/` (não apagar — virou memória de tentativas).

## Convenção de nome

`{nome-da-skill}.md` — começando com `/` removido. Exemplo: `gerar-checklist-aprovacao-mince.md`.

## Quando uma proposta amadurece?

Heurística: se o `/projeto-retrospectiva` detectar **rule-of-3** (o mesmo padrão aparece em 3 retrospectivas de projetos diferentes ou em deliverables diferentes do mesmo projeto), é forte sinal de promoção.
